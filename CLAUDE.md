# Liminote Daily - 工作守則

## 使用者
- 使用者不是工程師，不要假設他懂技術細節
- 用簡單的語言溝通，避免技術術語（必要時附上白話解釋）

## Git 與部署流程
- **所有開發一律在 `dev` 分支進行**，絕不直接在 `main` 上改東西
- 每次開始工作前，先確認自己在 `dev` 分支
- 完成功能/修復後，push 到 `dev` → Vercel 會自動產生測試站連結
- **主動告知使用者**：「已推到測試站，請到這個連結確認」
- 使用者確認沒問題後，再問：「測試沒問題的話，我幫你推到正式站？」
- 使用者同意後，才 merge `dev` → `main`（正式上線）
- 簡單說：dev = 測試站，main = 正式站，這個判斷由 Claude 負責

## Commit 規範
- 使用英文 commit message
- 格式：`Type: description`（如 `Fix:`, `Feat:`, `Refactor:`）
